02 — How Orion Scales

One Device → Many Devices → Many Agents

Three levels of execution: single device, cross-device via Firebase Real-Time DB relay, and parallel multi-agent orchestration.

2.1   Single Device

Every device runs a native Orion instance

Full agent loop with local tool access. 19 tools execute natively with full device privileges — not sandboxed VMs, not cloud screenshots.

2.2   Across Devices

Two modes. Mode A: remote tool execution on another device within the same conversation. Mode B: spawn an independent agent instance on the remote device with its own reasoning loop.

2.2a   Mode A — Direct Control

─────────────────────────                    ─────────────────────

Agent ─── wake_device("work-pc") ──────────>  Receives real-time DB signal
  |       |- verify ownership               Establishes WebSocket
  |       |- verify tier                     Metadata: skills,
  |       |- write wake signal               timezone, guardrails
  |       '- wait for connection  <──────────  Connected

Agent ─── Bash("git status", device="work-pc")
  |       |- lookup ws_session    ──────────>  execute natively
  |       '- receive result       <──────────  [text, images]
  |
  |       Result: "[Remote: Work PC] On branch main..."

Connection persists 2h (refreshed on each call).
Platform guards block incompatible tools (e.g., AppleScript on Windows).

2.4   Across Agents

Up to 5 parallel sub-agents per request, each running its own tool loop. A real-time broadcast channel lets agents share findings mid-run — auto-broadcast after every iteration, explicit peer-share on demand. Dependency graph controls ordering; failures can continue, abort, or skip.
